## 1. 计算机是怎么工作的？

想象你在餐厅点餐：

| 餐厅 | 计算机 |
|------|--------|
| 👨‍🍳 厨师 | **CPU（中央处理器）**，干活的人 |
| 📋 桌上的便签 | **内存（RAM）**，临时记东西 |
| 📦 仓库 | **硬盘**，永久存储 |
| 🍽️ 菜单 | **软件/程序**，告诉厨师怎么做 |

计算机的核心循环就是：**取指令 → 执行 → 取指令 → 执行 → ...**，每秒几十亿次。

## 2. 什么是编程？什么是 Python？

### 编程

编程就是**用计算机能理解的语言，写一份"菜谱"让它照着做**。

比如你写：
```python
print("你好，世界")
```

计算机就会在屏幕上显示 "你好，世界"。就这么简单。

### Python

Python 是一种编程语言——就像英语、中文一样，只是它是用来和计算机交流的。

**为什么叫 Python？** 创始人喜欢英国喜剧《蒙提·派森的飞行马戏团》（Monty Python），所以取名 Python（蟒蛇）。

**为什么选 Python？**
- 📝 语法接近英语，好读好写
- 🧩 有海量的"别人写好的代码"可以直接用
- 🌍 全球最流行的语言之一

```python
# 这段代码：如果分数大于60，打印"及格"，否则打印"不及格"
score = 75
if score > 60:
    print("及格")
else:
    print("不及格")
```

## 3. 什么是前端和后端？

还是餐厅比喻：

| 餐厅 | Web 应用 |
|------|----------|
| 🍽️ **大堂**（你看到的地方） | **前端**（浏览器里的界面） |
| 🔪 **后厨**（你看不到的地方） | **后端**（服务器上的程序） |
| 📝 服务员记菜名 | **API 请求**（前端告诉后端要什么） |
| 🍲 厨师做菜 | **后端处理**（计算、查数据库） |
| 🚶 服务员上菜 | **API 响应**（后端把结果送回前端） |

```
你的浏览器（前端）                   云端的服务器（后端）
     │                                      │
     │  "我要聊天，发了一句：你好"             │
     │─────────────────────────────────────→│
     │                                      │  查数据库
     │                                      │  调AI模型
     │                                      │
     │  "AI回复：你好！有什么可以帮你？"       │
     │←─────────────────────────────────────│
```

## 4. 什么是数据库？

**数据库 = 电子化的 Excel 表格 + 超级查询能力**

| Excel | 数据库 |
|-------|--------|
| 一个工作簿 | **数据库（Database）** |
| 一张 Sheet | **表（Table）** |
| 一行数据 | **一条记录（Row）** |
| 一列标题 | **字段（Column）** |

这个项目的数据库长这样：

```
📁 smart_assistant 数据库
├── 📄 conversations 表（会话表）
│   ├── id：会话编号
│   ├── title：会话标题
│   ├── role_id：使用的角色
│   └── model：使用的模型
├── 📄 messages 表（消息表）
│   ├── content：消息内容
│   ├── role：谁说的（user/assistant）
│   └── conversation_id：属于哪个会话
├── 📄 roles 表（角色表）
│   ├── name：角色名称
│   ├── prompt：角色提示词
│   └── icon：角色头像
└── 📄 users 表（用户表）
```

## 7. 这些技术分别干什么的？

### 7.1 Flask（Web 框架）

**比喻**：Flask 就像一个快递中转站。

```python
@chat_bp.route('/chat', methods=['POST'])   # 声明：收到 /chat 请求时
def chat():                                   # 执行这个函数
    data = request.get_json()                  # 拆开快递包裹
    result = ChatService.chat(...)             # 处理
    return success(result)                     # 打包发回去
```

### 7.2 LangChain + DeepSeek（大模型）

**比喻**：LangChain 是"万能遥控器"，DeepSeek 是"电视机"。

```python
# 不需要关心 DeepSeek 怎么工作的，LangChain 帮你搞定
model = ChatDeepSeek(api_key="你的密钥")
response = model.invoke("你好")    # 一键调用
print(response.content)            # "你好！有什么可以帮你？"
```

### 7.3 MySQL + SQLAlchemy（数据库）

**比喻**：SQLAlchemy 是"自动翻译机"，把 Python 代码翻译成 SQL 语句。

```python
# 用 Python 写：
conversation = Conversation.create(title="新对话")

# SQLAlchemy 自动翻译成 SQL：
# INSERT INTO conversations (id, title) VALUES ('...', '新对话');
```

### 7.4 HTML + CSS + JavaScript（前端三件套）

| 技术 | 比喻 | 在这个项目里 |
|------|------|-------------|
| **HTML** | 房子的骨架（墙在哪、门在哪） | 聊天框、按钮、输入框的位置 |
| **CSS** | 装修风格（颜色、大小、动画） | 深色主题、霓虹发光、粒子效果 |
| **JavaScript** | 家里的电器（让房子活起来） | 点击发送、接收消息、播放语音 |

### 7.5 SSE（Server-Sent Events）

**比喻**：普通请求是"点外卖等30分钟一次送到"，SSE 是"吃火锅，菜一份一份上"。

```
普通请求：  [发送] ────等待──── [一次返回全部]
SSE流式：   [发送] → 你 → 好 → ！ → 有 → 什 → 么 → ...
```

### 7.6 Redis（高速缓存）

**比喻**：Redis 是大脑里的"短期记忆"，超快但容量小。

在这个项目里，Redis 只做一件事：**数请求次数**。你一分钟请求超过60次？拒绝，防止滥用。

## 9. 一段代码是怎么运行的？

以 "你发一条消息，AI 回复" 为例：

### 第1步：你在浏览器里打字，点发送
```
浏览器 (JavaScript)：
  → 收集你的消息文字
  → 发送 POST 请求到 http://localhost:5000/api/v1/chat/stream
```

### 第2步：Flask 收到请求
```python
# app/api/chat.py 第34行
@chat_bp.route('/chat/stream', methods=['POST'])
def chat_stream():
    data = request.get_json()
    # data = {"message": "你好", "model": "deepseek", ...}
```

### 第3步：准备对话上下文
```python
# app/services/chat_service.py
conversation, memory, messages = ChatService.prepare_chat_context(...)
# → 加载历史消息 + 添加系统提示词 + 组装消息列表
```

### 第4步：调用 DeepSeek
```python
# app/llm/deepseek_client.py
for chunk in self.model.stream(messages):
    yield chunk.content   # 一个词一个词地返回
```

### 第5步：流式返回给浏览器
```python
# app/api/chat.py
for chunk in model_client.stream(messages):
    yield f"data: {chunk}\n\n"   # SSE 格式
```

### 第6步：浏览器逐字显示
```javascript
// static/index.html
bubble.innerHTML = renderMd(fullText);  // 实时渲染 Markdown
```

### 简化为图

```
[浏览器] → POST /api/v1/chat/stream → [Flask] → [ChatService] → [DeepSeek API]
                                                                       │
                                                                       ▼
[浏览器] ← SSE 流式数据 ←────────────────────────── [Flask] ← [AI 回复文字]
   │
   ├─ 实时打字显示
   ├─ Markdown 渲染
   └─ 语音朗读（可选）
```

## 10. 我该怎么修改这个项目？

### 10.1 我想换 AI 模型

编辑 `.env` 文件：
```env
# 把 deepseek 改成 qwen
DEFAULT_MODEL=qwen
```

### 10.2 我想改角色提示词

编辑 `app/__init__.py`，找到 `_seed_builtin_roles` 函数，改 prompt 文字，重启服务。

### 10.3 我想改颜色/主题

直接在浏览器里点 **🎨 主题** 按钮，或编辑 `static/index.html` 中的 `PRESETS` 对象。

### 10.4 我想加新功能

按这个顺序：
1. `app/models/` 加数据表（如果要存新数据）
2. `app/api/` 加新接口（暴露出 URL）
3. `app/services/` 加业务逻辑
4. `static/index.html` 加前端界面

### 10.5 改错了怎么办？

```bash
git checkout .          # 撤销所有改动
git log                 # 查看历史版本
git checkout <版本号>    # 回到某个版本
```

## 11. 常见名词速查表

| 名词 | 简单解释 |
|------|----------|
| **API** | 程序的"服务窗口"，你告诉它要什么，它给你结果 |
| **API Key** | 服务的"门禁卡"，证明你有权限使用 |
| **JSON** | 数据的"通用快递盒"，`{"key": "value"}` 格式 |
| **HTTP** | 浏览器和服务器之间的"通信语言" |
| **POST** | HTTP 的一种，"我要发数据给你" |
| **GET** | HTTP 的一种，"把数据给我看看" |
| **SSE** | 服务器主动推数据，"菜一份份上" |
| **Token** | AI 模型计数单位（约等于一个汉字或一个英文单词） |
| **Prompt** | 提示词，告诉 AI "你现在扮演什么角色" |
| **Framework** | 框架，一套帮你省事的"预制板房" |
| **Library** | 库，别人写好的工具，"拿来就用" |
| **ORM** | 对象关系映射，用 Python 操作数据库，不用写 SQL |
| **CORS** | 跨域资源共享，允许网页从不同地址请求数据 |
| **WSGI** | Python Web 应用和服务器之间的"标准接口" |
| **Git** | 版本控制，代码的"时光机"（存档、回退、协作） |
| **GitHub** | 代码托管网站，"代码的百度网盘" |
| **Open Source** | 开源，代码公开，谁都能看、谁都能用 |
